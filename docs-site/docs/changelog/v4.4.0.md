# v4.4.0 — Auditoría completa de trazabilidad

> Fecha: 2026-03-28 | Rama: panel-v2 | Riesgo máximo: ALTO

## Resumen

Auditoría profunda del sistema de trazabilidad de contactos. La cobertura de logging pasó de ~26% a ~85%, agregando `usuario_id` a todos los logs existentes, cerrando brechas críticas en la creación de contactos, multimedia, notas, finanzas, merge y configuración de embudo.

## Cambios

### feat
- Logging completo en `actionsSlice`: notas (update con diff, delete con contactoId), multimedia (upload/delete), merge visible en timeline del contacto primario (STATE)
- Logging completo en `financeStore`: servicios CRUD y pagos CRUD con entidad_tipo y datos del cambio (STATE)
- Logging completo en `funnelSlice`: crear, actualizar, eliminar y reordenar etapas de embudo con diff antes/después (STATE)
- Diff visual en pause/reactivate de contactos con datos_antes/datos_despues (STATE)
- Index SQL compuesto para performance del timeline: `(empresa_id, contacto_id, fecha_creacion DESC)` (DATABASE)

### fix
- Agregado `contactoId` faltante en logs de eliminar nota, actualizar/eliminar asignación — ahora aparecen en el timeline del contacto (STATE)

### docs
- Simplificación del tema VitePress: eliminación de modo usuario, consolidación a modo equipo único (DOCS)
