# v4.2.0 — Mora operativa, WhatsApp resiliente, trazabilidad

> Fecha: 2026-03-28 | Rama: panel-v2 | Riesgo maximo: ALTO

## Resumen

Corrige la vista de cartera para mostrar la mora del servicio con actividad de pago mas reciente (en vez del peor caso). Mejora la resiliencia del envio de mensajes WhatsApp y agrega trazabilidad de usuario en el registro de actividades.

## Cambios

### feat
- Vista lista de cartera muestra mora operativa del servicio con menor atraso positivo (`displayItem`) en vez del mas critico (API)
- Activity logger ahora registra `usuarioId` del operador en todas las acciones de contacto (STATE)
- Log de creacion de contacto en el registro de actividades (STATE)
- Soporte para metrica INP (Interaction to Next Paint) en Web Vitals (UI)
- Circuit breaker de engagement con auto-reset tras 5 minutos (API)

### fix
- WhatsApp: `sendDirectMessageRecord` ahora espera la respuesta de la API interna y retorna error explicito si falla, en vez de fire-and-forget silencioso (API)
- Token de webhook hardcodeado reemplazado por variable de entorno `WEBHOOK_AUTH_TOKEN` (API)
- `logApiError` consolidado para usar `logError` con spike tracking (API)
- Reemplazo de `as any` por `NotificationStatus` tipado en notificationsStore (STATE)

### refactor
- ContactDetailPanel: elimina query HITL duplicada, delega conteo al hijo `ContactQueriesView` via callback (UI)
- ContactQueriesView: unifica suscripcion realtime en el padre, agrega ARIA labels y motion-reduce (UI)
- ObservabilityDashboard: elimina RPCs de engagement (schema opcional), compara periodos equivalentes, uptime real del servidor (UI)
- Health endpoint usa lazy singleton de Supabase en vez de crear cliente por request (API)

### docs
- Reescritura completa de documentacion de modulos para modo dual equipo/usuario (DOCS)
- Tema VitePress con Layout dual, ModeSwitcher, componentes TeamHome/UserHome (DOCS)
