## 🔐 Sistema de Logging

### Tablas de Base de Datos

#### `wp_error_logs`
Registro centralizado de errores de aplicación.

```sql
CREATE TABLE wp_error_logs (
  id BIGSERIAL PRIMARY KEY,
  function_name TEXT NOT NULL,
  error_message TEXT,
  error_stack TEXT,
  request_body TEXT,
  user_id UUID,
  empresa_id BIGINT,
  severity TEXT CHECK (severity IN ('debug', 'info', 'warning', 'error', 'critical')),
  context JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Índices optimizados:**
- `idx_wp_error_logs_created_at` - Búsquedas por fecha
- `idx_wp_error_logs_function_name` - Búsquedas por función
- `idx_wp_error_logs_severity` - Filtrado por severidad
- `idx_wp_error_logs_empresa_id` - Multi-tenant filtering

### Uso del Error Logger

```typescript
import { logError, logWarning, logCritical, logApiError } from '@/lib/error-logger';

// Log de error genérico
try {
  await fetchContacts();
} catch (error) {
  await logError('fetchContacts', error, {
    userId: user.id,
    empresaId: empresa.id
  });
}

// Log de error crítico
await logCritical('paymentProcessing', error, {
  empresaId: empresa.id,
  additionalData: { amount, currency }
});

// Log de warning
await logWarning('slowQuery', 'Query took 5s', {
  additionalData: { queryName: 'fetchAllContacts' }
});

// Log de error de API
await logApiError('/api/contacts', error, requestBody, {
  userId: user.id
});
```

### Niveles de Severidad

| Nivel | Uso | Acción Requerida |
|-------|-----|------------------|
| `debug` | Información de desarrollo | Ninguna |
| `info` | Eventos informativos | Ninguna |
| `warning` | Situaciones anómalas | Revisar |
| `error` | Errores recuperables | Investigar |
| `critical` | Errores críticos | Acción inmediata |

## 📊 Sistema de Auditoría

### Tabla `wp_actividades_log`

Registro completo de actividades del sistema para compliance y debugging.

```sql
CREATE TABLE wp_actividades_log (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  tipo TEXT NOT NULL,
  accion TEXT NOT NULL,
  descripcion TEXT,
  agente_id BIGINT,
  empresa_id BIGINT,
  contacto_id BIGINT,
  entidad_tipo TEXT,
  datos_antes JSONB,
  datos_despues JSONB,
  fecha_creacion TIMESTAMPTZ DEFAULT NOW(),
  usuario_id UUID,
  entidad_id TEXT,
  ip_origen TEXT,
  user_agent TEXT,
  tipo_valido TEXT CHECK (tipo_valido IN ('contacto', 'cita', 'conversacion', ...))
);
```

**Índices optimizados:**
- `idx_actividades_empresa_fecha` - Queries por empresa y fecha
- `idx_wp_actividades_log_tipo_accion` - Búsquedas por tipo/acción
- `idx_wp_actividades_log_entidad` - Timeline de entidades

### Uso del Activity Logger

```typescript
import { 
  logActivity, 
  logContactActivity, 
  logAuth,
  logCreate,
  logUpdate,
  logDelete 
} from '@/lib/activity-logger';

// Log de actividad genérica
await logActivity({
  tipo: 'contacto',
  accion: 'actualizar',
  descripcion: 'Actualizado estado del contacto',
  contactoId: 123,
  empresaId: 1,
  datosAntes: { estado: 'nuevo' },
  datosDespues: { estado: 'calificado' }
});

// Log de creación
await logCreate('tarea', tareaId, tareaData, {
  empresaId: 1,
  contactoId: 123
});

// Log de actualización
await logUpdate('cita', citaId, oldData, newData, {
  empresaId: 1
});

// Log de autenticación
await logAuth('login', userId, {
  ipOrigen: req.ip,
  userAgent: req.headers['user-agent']
});

// Log de actividad de contacto
await logContactActivity('actualizar', contactoId, empresaId, 
  'Estado cambiado a calificado',
  { antes: { estado: 'nuevo' }, despues: { estado: 'calificado' } }
);
```

### Tipos de Actividades

- `contacto` - Operaciones sobre contactos
- `cita` - Gestión de citas
- `conversacion` - Interacciones de chat
- `tarea` - Gestión de tareas
- `proyecto` - Gestión de proyectos
- `campana` - Campañas de marketing
- `email` - Envío/recepción de emails
- `nota` - Notas de contacto
- `auth` - Autenticación
- `admin` - Acciones administrativas
- `sistema` - Eventos del sistema

## 📈 Monitoreo de Performance

### Métricas Disponibles

```typescript
import { 
  trackMetric, 
  trackQuery, 
  trackRender,
  trackWebVital,
  getMetricSummary,
  logPerformanceReport 
} from '@/lib/performance-monitor';
```

### Track Custom Metrics

```typescript
// Métrica personalizada
trackMetric('contacts_load_time', 1250, 'ms', {
  count: 100,
  filter: 'calificados'
});

// Track Supabase query
const contacts = await trackQuery('fetchContacts', async () => {
  return await supabase.from('wp_contactos').select('*');
});

// Track component render
trackRender('ContactList', 45.2);

// Track Web Vitals
trackWebVital('LCP', 2300);
```

### Análisis de Performance

```typescript
// Resumen de métrica
const summary = getMetricSummary('contacts_load_time');
// {
//   count: 50,
//   avg: 1200,
//   min: 800,
//   max: 2500,
//   p50: 1150,
//   p95: 2100,
//   p99: 2400
// }

// Reporte completo
await logPerformanceReport();
```

### Umbrales de Alerta

El sistema automáticamente logea warnings cuando:

- **Métricas > 3000ms** - Operación lenta
- **Queries > 2000ms** - Query lenta
- **Renders > 100ms** - Render lento
- **Re-renders > 50** - Exceso de re-renders
- **Web Vitals "poor"** - Métricas por debajo del estándar

## 🎯 Mejores Prácticas

### 1. Logging Estructurado

```typescript
// ❌ Evitar
console.error('Error:', error);

// ✅ Usar
await logError('functionName', error, {
  userId: user.id,
  empresaId: empresa.id,
  additionalData: { context: 'important' }
});
```

### 2. Validación de Inputs

```typescript
// ❌ Evitar
const { message } = await req.json();

// ✅ Usar
const validation = await validateApiRequest(req, MonicaRequestSchema);
if (!validation.valid) return validation.response;
const { message } = validation.data;
```

### 3. Tracking de Performance

```typescript
// ❌ Evitar
const data = await fetchData();

// ✅ Usar
const data = await trackQuery('fetchData', async () => {
  return await fetchData();
});
```

### 4. Error Boundaries

```tsx
{/* ❌ Evitar */}
<CriticalComponent />

{/* ✅ Usar */}
<ErrorBoundary>
  <CriticalComponent />
</ErrorBoundary>
```

### 5. Auditoría de Cambios

```typescript
// ❌ Evitar
await supabase.from('wp_contactos').update(newData).eq('id', id);

// ✅ Usar
const oldData = await getContact(id);
await supabase.from('wp_contactos').update(newData).eq('id', id);
await logUpdate('contacto', id, oldData, newData, { empresaId });
```

## 🔒 Seguridad Multi-Tenant

### Row Level Security (RLS)

Todas las tablas de logging tienen RLS habilitado:

```sql
-- Los usuarios solo ven logs de su empresa
CREATE POLICY "Users can view error logs from their empresa"
  ON wp_error_logs FOR SELECT
  USING (
    empresa_id IS NULL OR
    empresa_id IN (SELECT unnest(get_user_empresa_ids()))
  );
```

### Función Helper

```sql
-- Retorna los IDs de empresa del usuario autenticado
get_user_empresa_ids() RETURNS BIGINT[]
```

### Monitoreo de Tamaño

```sql
-- Verificar tamaño de tablas
SELECT 
  schemaname,
  tablename,
  pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE tablename IN ('wp_error_logs', 'wp_actividades_log')
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC;
```
